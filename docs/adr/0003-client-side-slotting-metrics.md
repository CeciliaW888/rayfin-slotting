# Compute slotting metrics in the browser, not a backend solver

Travel, ABC class, golden-zone compliance, the slotting health score, and the Phase-3 optimizer all run client-side in the React/R3F app, reading Slots and SKUs through the Rayfin typed client. The dataset is small (tens of aisles, ~hundreds of SKUs) and the formulas are simple and explainable (`travel = Σ pick-rate × distance-to-dock`), so a browser computation is fast, fully auditable, and needs no server code. Trade-off accepted: this won't scale to a real multi-thousand-SKU DC or a sophisticated multi-constraint solver — if we ever need that, the computation moves to a Rayfin function/UDF or a backend job. Recorded because a future reader may expect a backend optimizer and wonder why the math lives in the frontend; this is deliberate for a demo/evaluation tool that prizes transparency over optimization depth.
