# Position as a Fabric-native evaluation tool, not an the commercial suite replacement

The company is evaluating procuring the vendor/the vendor the commercial suite DC, a mature multi-constraint slotting optimizer. We deliberately do NOT try to match its optimization depth (dimensions/weight/crushability/pallet-build/affinity/seasonality, WMS integration, validated ROI, professional services). Instead the Slotting Digital Twin is scoped as a Fabric-native, interactive 3D *evaluation and business-case* tool that quantifies the slotting opportunity on the company's own data and covers the high-value 80% case (velocity/ABC + travel-distance + what-if + a transparent optimize), with a path toward differentiated capabilities the incumbent suites lack (web 3D accessibility, explainability, AI copilot, real-time via Fabric). This is a scope/boundary decision (the explicit "no" to feature-parity is as important as the "yes") made to avoid over-claiming in a procurement conversation; it can be revisited if leadership decides to invest in a full in-house optimizer. Default positioning is "evaluation tool now, evolving toward differentiated competitor" — to be confirmed with the business owner.
